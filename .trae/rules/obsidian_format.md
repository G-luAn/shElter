# Obsidian 格式规范 / Obsidian Format Rules

## 核心原则

所有文件必须遵循 Obsidian 格式要求。

---

## 转义符禁用

**禁止使用转义符**。直接使用原始 Markdown 符号，让 Obsidian 正常解析。

### 禁用的转义符

- `\*` — 应使用 `*`
- `\#` — 应使用 `#`
- `\$` — 应使用 `$`
- `\!` — 应使用 `!`
- `\[` `\]` — 应使用 `[` `]`
- `\(` `\)` — 应使用 `(` `)`
- 其他所有以 `\` 开头的转义符

### 正确 vs 错误示例

错误（使用转义符）：
```
\*这是斜体\*
\# 这是标题\#
```

正确（直接使用符号）：
```
*这是斜体*
# 这是标题
```

## Obsidian 标签系统

Obsidian 的标签（Tags）是连接笔记的桥梁，必须正确使用。

### 标签格式

- 标签以 `#` 开头，如 `#标签名`
- 标签不能包含空格
- 可以使用连字符 `-` 或下划线 `_` 连接词
- 标签可以放在文件末尾或相关章节处

### 正确标签示例

```
#冥想 #每日功课 #概念解释
```

### 错误标签示例

```
# 冥想     ← 有空格
#冥想功课  ← 难以识别边界
```

## Frontmatter（YAML 头栏）

如果文件需要 Frontmatter，使用标准的 YAML 格式。

### Frontmatter 格式

```
---
标签:
  - I
  - II
  - 冥想
创建日期: 2024-01-01
---

正文内容...
```

### Frontmatter 注意事项

- 使用 `---` 包裹，放在文件最开头
- 标签在 Frontmatter 中使用列表格式
- 冒号后有空格，格式整洁
- 不要在 Frontmatter 中使用转义符

## Markdown 基本格式

### 标题

```
# 一级标题
## 二级标题
### 三级标题
```

### 列表

```
- 无序列表项
- 另一个列表项

1. 有序列表项
2. 另一个有序项
```

### 引用

```
> 这是引用内容
```

### 代码块

````
```语言
代码内容
```
````

## 文件结构建议

建议的文件结构：

```
---
标签:
  - #I
---

# 笔记标题

正文内容...

---

#相关标签 #五界
```

---

## 跨平台兼容性

- 文件名避免特殊字符（除 `-` 和 `_` 外）
- 路径分隔符在 Windows 下使用 `\`，但在文件名中避免使用
- 保持文件名简洁清晰
